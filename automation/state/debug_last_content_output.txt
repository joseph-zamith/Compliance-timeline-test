    <div style="margin-bottom: 28px; padding-bottom: 20px; border-bottom: 1px solid #e9ebee;">
        <h3 style="color: #ff512c; font-size: 16px; font-weight: 700; margin: 0 0 12px 0;">ISO/TS 24971-2 : Guidance Machine Learning dans gestion des risques</h3>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Date publication :</strong> 9 juillet 2026 (ISO)</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Contenu :</strong> ISO/TS 24971-2 constitue complément normatif ISO 14971:2019 (gestion des risques DM). Document structure approche risque pour algorithmes ML/IA : cadre validation modèles, détection drift performances, gestion biais/fairness, traçabilité décisions algorithmiques. Alignement explicite directives MDR articles 17-18 (risque) et AI Act Annexe III (haut risque). Cible : logiciels IA classe II/III, SaMD.</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Pourquoi ça change :</strong> Jusqu'à présent, cadre risque ML restait flou—ISO 14971 ne traitait pas spécifiquement ML, MDCG 2025-6 donnait cadre réglementaire mais pas méthodologie risque. ISO/TS 24971-2 propose matrice structurée : 1) risque domaine (diagnostic/prédiction), 2) risque entrée données (biais amont), 3) risque modèle (drift, adversarial), 4) risque déploiement (efficacité réelle vs attendu). MDCG évalue intégration normes harmonisées EN ISO 13485:2016/A11 (QMS) + EN 62304/A1 (cycle logiciel) Q4 2026—ISO/TS 24971-2 sera référence implémentation.</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Deadline/action :</strong> Impact dossiers en cours classe II/III IA :
        <ul style="color: #1c2837; font-size: 12px; margin: 8px 0 0 20px; padding-left: 20px;">
            <li>Court terme (août-septembre 2026) : auditez matrice risque ML dossiers en phase soumission notifié—vérifiez documentation validation données + drift detection.</li>
            <li>Moyen terme (T4 2026) : attendez MDCG guidance harmonisation ISO/TS 24971-2 EN 13485/A11/A1, puis intégrez au dossier technique permanent.</li>
            <li>Compliance interne : formez QARA/R&D ISO/TS 24971-2 cadre sept 2026 avant soumission nouveaux dossiers.</li>
        </ul></p>
        <p style="color: #ff512c; font-size: 11px; margin: 12px 0 0 0;"><a href="https://www.qualitiso.com/veille/" style="color: #ff512c; text-decoration: none;">Source: Qualitiso</a></p>
    </div>

    <div style="margin-bottom: 28px; padding-bottom: 20px; border-bottom: 1px solid #e9ebee;">
        <h3 style="color: #ff512c; font-size: 16px; font-weight: 700; margin: 0 0 12px 0;">Décision d'exécution (UE) 2026/1231 : sept normes harmonisées entrent en vigueur 1er août 2026</h3>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Date publication :</strong> 18 juin 2026 (Journal officiel UE). <strong>Entrer en vigueur :</strong> 1er août 2026.</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Contenu :</strong> Commission européenne publie décision d'exécution listant normes harmonisées révisées/nouvelles applicables MDR à partir 1er août 2026. Énumération clés pour MDSW :
        <ul style="color: #1c2837; font-size: 12px; margin: 8px 0 0 20px; padding-left: 20px;">
            <li>EN ISO 13485:2016/A11 : QMS—intégration management IA, traçabilité décisions critique.</li>
            <li>EN 62304/A1 : cycle logiciel—révision conformité AI Act (validation adversarial, gestion drift).</li>
            <li>EN 62366-1:2015+A1:2020 : usabilité—impact IHM IA (clarté output algorithme).</li>
            <li>IEC 82304-1:2018 : sécurité santé digitale—nouvelles exigences cybersécurité, gestion incident.</li>
            <li>ISO 15223-1/-2, ISO 20417 : symboles/étiquetage—alignement nomenclature UE.</li>
        </ul></p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Pourquoi ça change :</strong> Jusqu'en juillet 2026, certaines entreprises pouvaient appliquer versions antérieures normes si démonstration conformité raisonnable. Dès 1er août, conformité EN 13485:2016/A11 + EN 62304/A1 devient obligatoire nouveau dépôt auprès organisme notifié—refus automatique dossiers en ancienne version sans justification transitoire (rare). EN 13485:2016/A11 spécifique IA introduit concept « AI impact assessment » obligatoire prise de décision critique; EN 62304/A1 impose validation robustesse algorithme contre perturbations adversarial. Pour logiciels classe II/III, conformité immédiate exigée.</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Deadline/action :</strong>
        <ul style="color: #1c2837; font-size: 12px; margin: 8px 0 0 20px; padding-left: 20px;">
            <li><strong>Urgent (fin juillet) :</strong> dossiers en phase soumission notifié avant 1er août doivent certifier conformité Décision 2026/1231—audit complet classification + dossier technique recommandé.</li>
            <li><strong>Court terme (août) :</strong> refus systématique dossiers non-conformes EN 13485:2016/A11 + EN 62304/A1—rejet saisine automatique notifié, délai révision 4-8 semaines.</li>
            <li><strong>Documentation :</strong> créez registre « Impact Décision 2026/1231 » par produit : classification, normes applicables, audit conformité, délai implémentation.</li>
        </ul></p>
        <p style="color: #ff512c; font-size: 11px; margin: 12px 0 0 0;"><a href="https://www.qualitiso.com/veille/" style="color: #ff512c; text-decoration: none;">Source: Qualitiso / Commission européenne</a></p>

    <div style="margin-bottom: 28px; padding-bottom: 20px; border-bottom: 1px solid #e9ebee;">
        <h3 style="color: #ff512c; font-size: 16px; font-weight: 700; margin: 0 0 12px 0;">MDCG 2026-4 : Management SS(C)P dans EUDAMED</h3>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Date publication :</strong> 18 juin 2026 (MDCG). <strong>Entrée en vigueur :</strong> immédiate pour EUDAMED.</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Contenu :</strong> Commission publie guide MDCG 2026-4 précisant gestion Surgical Site Contamination Precautions (SS(C)P) portefeuille EUDAMED. Nouveauté majeure : traçabilité complète SS(C)P devenue obligatoire audit-trail système, responsabilité administrateur EUDAMED étendue vérification chaîne conformité. Document définit : 1) qui déclare SS(C)P (mandataire/fabricant), 2) quand (dès notification), 3) comment (formulaire structures EUDAMED + justificatifs), 4) qui vérifie (ANSM/organismes notifiés accès audit-trail). Cas non-conformité = blocage dossier EUDAMED, signalement automatique autorité.</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Pourquoi ça change :</strong> Jusqu'en juin 2026, gestion SS(C)P restait responsabilité décentralisée mandataire/fabricant—audit-trail EUDAMED n'était pas automatisé, vérification autorité était sporadique. MDCG 2026-4 introduit chaîne responsibility formelle : chaque modification SS(C)P génère événement EUDAMED horodaté, traçabilité irréversible. Cible : prévenir fraude SS(C)P (faux certificats), renforcer contrôle ANSM. Charge administrateurs EUDAMED augmente significativement : vérification exhaustive SS(C)P dossiers, documentation justificatifs, coordin ANSM.</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Deadline/action :</strong>
        <ul style="color: #1c2837; font-size: 12px; margin: 8px 0 0 20px; padding-left: 20px;">
            <li><strong>Immédiat (juillet-août) :</strong> auditez portefeuille EUDAMED configuration SS(C)P tous produits—vérifiez conformité justificatifs vs déclaration.</li>
            <li><strong>Q3 2026 :</strong> documenter toute modification SS(C)P avec justificatif conforme MDCG 2026-4; formez équipe administrateurs EUDAMED processus nouveau.</li>
            <li><strong>Risque :</strong> dossiers SS(C)P non-conformes = rejet automatique EUDAMED, délai correction 2-4 semaines.</li>
        </ul></p>
        <p style="color: #ff512c; font-size: 11px; margin: 12px 0 0 0;"><a href="https://www.qualitiso.com/veille/" style="color: #ff512c; text-decoration: none;">Source: Qualitiso</a></p>

    <div style="margin-bottom: 28px; padding-bottom: 20px; border-bottom: 1px solid #e9ebee;">
        <h3 style="color: #ff512c; font-size: 16px; font-weight: 700; margin: 0 0 12px 0;">MDCG 2021-5 Annexe : Transition symbole mandataire « EC REP » vers « EU REP »</h3>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Date publication :</strong> 18 juin 2026 (MDCG). <strong>Délai transition :</strong> septembre 2026 (nouveaux) / décembre 2026 (portefeuille existant).</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Contenu :</strong> Commission actualise annexe MDCG 2021-5 (symboles/marquage DM) pour migration nomenclature « mandataire autorisé ». Changement : symbole ancien « EC REP » (European Community Representative) remplacé « EU REP » (European Union Representative) aligner nomenclature post-Brexit. Symbole apply marquage produit, étiquetage, dossier EUDAMED. Transition graduée : septembre 2026 symbole nouveau obligatoire enregistrements neufs; décembre 2026 tous produits portefeuille existant doivent afficher « EU REP ».</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Pourquoi ça change :</strong> Migration symbolique UK hors UE (Brexit) nécessitait harmonisation nomenclature—« EC REP » impliait UK membre (obsolète); « EU REP » clarifie mandataire UE-only. Changement cosmétique mais impact conformité réglementaire : dossiers EUDAMED rejetés si symbole obsolète, risque pénalités ANSM si marquage produit non-conforme décembre 2026.</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Deadline/action :</strong>
        <ul style="color: #1c2837; font-size: 12px; margin: 8px 0 0 20px; padding-left: 20px;">
            <li><strong>Septembre 2026 :</strong> enregistrements neufs EUDAMED obligatoirement symbole « EU REP »; ancien rejeté.</li>
            <li><strong>Octobre-novembre 2026 :</strong> auditez portefeuille produits marquage « EC REP »—identifiez stocks/emballages à renouveler avant décembre.</li>
            <li><strong>Décembre 2026 :</strong> tous produits dossier EUDAMED, marquage physique, étiquetage doivent afficher « EU REP ».</li>
            <li><strong>Gestion stock :</strong> coordonnez qualité/logistique rotation produits « EC REP » existants avant transition complète.</li>
        </ul></p>
        <p style="color: #ff512c; font-size: 11px; margin: 12px 0 0 0;"><a href="https://www.qualitiso.com/veille/" style="color: #ff512c; text-decoration: none;">Source: Qualitiso</a></p>

        <h3 style="color: #ff512c; font-size: 16px; font-weight: 700; margin: 0 0 12px 0;">MDCG 2021-24 Rev.1 : Mise à jour classification DM (avril 2026)</h3>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Date publication :</strong> 28 avril 2026 (MDCG). <strong>Applicabilité :</strong> immédiate.</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Contenu :</strong> Commission publie révision 1 guide MDCG 2021-24 (classification DM MDR) intégrant clarifications AI Act + jurisprudence 18 mois. Changements clés : 1) IA-as-a-Service classification—nouvel arbre décisionnel distingue IA infrastructure (classe I) vs IA diagnostic/prédiction critique (classe III); 2) accessoires : affinage règles accessoires DM IA (clarté algorithme à patient = pas accessoire); 3) cas d'usage : diagnostic imagerie IA, prédiction risque événement grave, recommandation traitement = classe II minimum. Document inclus exemples concrets.</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Pourquoi ça change :</strong> Jusqu'en avril 2026, classification logiciels IA restait imprécise—certains soumettaient « infrastructure IA » (classe I), autorités notifiées objectaient (classe II/III) créant délais dépôt. MDCG 2021-24 Rev.1 clarifie : contexte utilisation détermine classe, pas technologie. Arbre décisionnel explicite épargne allers-retours 6-8 semaines.</p>
        <p style="color: #1c2837; font-size: 13px; line-height: 1.7; margin: 0 0 12px 0;"><strong>Deadline/action :</strong>
        <ul style="color: #1c2837; font-size: 12px; margin: 8px 0 0 20px; padding-left: 20px;">
            <li><strong>Avant soumission notifié (Q3 2026) :</strong> vérifiez classification dossiers en cours vs MDCG 2021-24 Rev.1—décalage = rejet soumission.</li>
            <li><strong>Portefeuille existant :</strong> auditez classification produits approuvés—si écart important (classe II approuvé, Rev.1 dit classe III), évaluez requalification.</li>
        </ul></p>
