<h2 style="margin:0 0 20px 0; font-size:18px; color:#1c2837; border-bottom:3px solid #ff512c; padding-bottom:8px;">UE et international</h2>

<h3 style="margin:24px 0 12px 0; font-size:15px; color:#1c2837; font-weight:600;">CNIL — Méthodologies MR-001 & MR-003 (23 mai 2026)</h3>
<p style="margin:0 0 8px 0; font-size:13px; color:#213045; line-height:1.6;">
Modernisation du cadre RGPD pour la recherche en santé. Les deux nouvelles méthodologies remplacent des guides antérieurs et précisent :
</p>
<ul style="margin:8px 0 0 0; padding-left:20px; font-size:13px; color:#213045; line-height:1.6;">
<li>MR-001 : minimisation et finalité dans les recherches épidémiologiques et cliniques.</li>
<li>MR-003 : critères de pseudonymisation durables et droits des données en recherche longitudinale.</li>
</ul>
<p style="margin:12px 0 0 0; font-size:12px; color:#ff512c; font-weight:600;">
Impact : audit immédiat recommandé pour tout acteur effectuant de la recherche multi-centre ou utilisant des données de santé de grande ampleur. Délai conseillé : avant fin août.
</p>
<p style="margin:8px 0 0 0; font-size:12px;"><a href="https://www.cnil.fr/fr" style="color:#ff512c; text-decoration:none; font-weight:600;">Source : CNIL →</a></p>

<h3 style="margin:24px 0 12px 0; font-size:15px; color:#1c2837; font-weight:600;">Suisse — Swissdamed en vigueur (1er juillet 2026)</h3>
<p style="margin:0 0 8px 0; font-size:13px; color:#213045; line-height:1.6;">
Entrée en application de l'obligation d'enregistrement dans Swissdamed. Concerne tous les DM commercialisés en Suisse, tout classe. Swissmedic n'accepte plus les enregistrements parallèles en papier.
</p>
<p style="margin:12px 0 0 0; font-size:12px; color:#ff512c; font-weight:600;">
Action urgente : vérifier que tous les produits sont enregistrés. Non-conformité après 31 juillet entraine retrait du marché automatique. Délai pour régularisation : très court.
</p>
<p style="margin:8px 0 0 0; font-size:12px;"><a href="https://www.swissmedic.admin.ch" style="color:#ff512c; text-decoration:none; font-weight:600;">Source : Swissmedic →</a></p>

<h3 style="margin:24px 0 12px 0; font-size:15px; color:#1c2837; font-weight:600;">Australie — Registres de traçabilité (1er juillet 2026)</h3>
<p style="margin:0 0 8px 0; font-size:13px; color:#213045; line-height:1.6;">
Nouvelle obligation TGA : enregistrement des numéros de lot, numéros de lot exportateur, dates d'expiration et destinataires finaux. Système électronique obligatoire. Applique à tous les DM classe IIa et supérieures.
</p>
<p style="margin:12px 0 0 0; font-size:12px; color:#ff512c; font-weight:600;">
Impact : exigence parallèle à Eudamed. Les fabricants doivent intégrer ce registre dans leur système de traçabilité.
</p>
<p style="margin:8px 0 0 0; font-size:12px;"><a href="https://www.tga.gov.au/devices-tracking" style="color:#ff512c; text-decoration:none; font-weight:600;">Source : TGA →</a></p>

<h3 style="margin:24px 0 12px 0; font-size:15px; color:#1c2837; font-weight:600;">UK MHRA — Rapport AI Airlock Phase 2 (juin 2026)</h3>
<p style="margin:0 0 8px 0; font-size:13px; color:#213045; line-height:1.6;">
Le programme pilote a testé l'évaluation de 12 logiciels IA. Le rapport synthétise : critères d'audit adapté à l'IA, risques liés au cycle d'entraînement continu, et préconisations pour une norme post-Brexit. Alignement futur avec l'AI Act européen discuté.
</p>
<p style="margin:8px 0 0 0; font-size:12px;"><a href="https://www.mhra.gov.uk/publications/ai-airlock-phase-2" style="color:#ff512c; text-decoration:none; font-weight:600;">Source : MHRA →</a></p>

<h3 style="margin:24px 0 12px 0; font-size:15px; color:#1c2837; font-weight:600;">MDCG 2026-4 — SSCP dans Eudamed (18 juin 2026)</h3>
<p style="margin:0 0 8px 0; font-size:13px; color:#213045; line-height:1.6;">
Document de guidance sur l'enregistrement des suppressions, suspensions et changements de statut de conformité progressive. Précise : quand notifier (préalablement ou lors du changement), délais de publication, et interactions avec Eudamed et NANDO.
</p>
<p style="margin:8px 0 0 0; font-size:12px;"><a href="https://health.ec.europa.eu/documents/mdcg-2026-4_en" style="color:#ff512c; text-decoration:none; font-weight:600;">Source : Commission EU →</a></p>

<h3 style="margin:24px 0 12px 0; font-size:15px; color:#1c2837; font-weight:600;">Normes harmonisées & EMDN v2026 (avril-juin 2026)</h3>
<p style="margin:0 0 8px 0; font-size:13px; color:#213045; line-height:1.6;">
MDCG 2026-1, 2, 3 publiés : mise à jour majeure de la nomenclature EMDN, ajout de codes pour de nouvelles catégories DM (ex. logiciels connectés, implants biocompatibles novotech), et révision des critères de classification. MDCG 2021-24 Rev.1 également mise à jour (avril 2026) avec clarifications sur les DM en tant que service (SaMD et logiciels d'aide à la décision).
</p>
<p style="margin:8px 0 0 0; font-size:12px;"><a href="https://www.qualitiso.com/veille" style="color:#ff512c; text-decoration:none; font-weight:600;">Source : Qualitiso →</a></p>

Rapport MHRA AI Airlock et suivi UK DTPR (Device Type Registration) continue en parallèle de Swissdamed. Aucune autre évolution réglementaire majeure cette semaine.
